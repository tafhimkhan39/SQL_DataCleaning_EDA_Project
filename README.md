# SQL_DataCleaning_EDA_Project

## Introduction

This project demonstrates the use of **MySQL** to clean, explore, and analyze data stored in a relational database. The goal of the project was to gain hands-on experience with SQL by working with a structured dataset and performing common data analysis tasks such as filtering data, aggregating results, and combining tables using joins.

SQL is one of the most essential tools used by **data analysts, data scientists, and business analysts** to query and manipulate data stored in databases. This project helped strengthen my understanding of how relational databases function and how SQL queries can be used to extract meaningful insights from data.

---

## Overview

In this project, a sample relational dataset was imported into **MySQL Workbench** and analyzed using SQL queries.

The project focuses on cleaning the database structure, retrieving specific data, analyzing patterns using aggregate functions, and combining tables through joins. These techniques are commonly used in real-world data analysis workflows.

The project was completed by following a two-part SQL tutorial and implementing the queries within a local MySQL environment.

---

## Tools Used

* **MySQL** – Relational database management system used to store and query data
* **MySQL Workbench** – Interface used to manage the database and run SQL queries
* **SQL (Structured Query Language)** – Used to retrieve, filter, aggregate, and analyze the dataset

---

## Data Cleaning process
1) Remove Duplicates
2)  Standardize the Data
3)  Handle Null Values or blank values
4)  Remove Any Columns

## Data Analysis

Gathered useful information such as
